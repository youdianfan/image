# MVP 开发计划（基于 Electron + Vue 3 方案）

## 版本规划总览

```
V1 (MVP)  ——  2-3 周  ——  批量重命名 + 格式转换 + 压缩
V2        ——  2 周    ——  AI离线命名 + SEO优化
V3        ——  3 周    ——  图片内容识别（CV模型）
```

---

## V1 MVP 详细开发计划

### 第一阶段：项目搭建（2天）

**Day 1: 环境初始化**

- [ ] 用 electron-vite 创建项目 `npm create @quick-start/electron`
- [ ] 配置 TypeScript、ESLint、Prettier
- [ ] 安装核心依赖：Vue 3、Pinia、Element Plus、Vue Router
- [ ] 配置 electron-builder 打包（Win + Mac）
- [ ] 验证 dev 模式和打包流程跑通

**Day 2: 基础架构**

- [ ] 搭建主进程 ↔ 渲染进程 IPC 通信框架
- [ ] 编写 preload.ts 安全桥接
- [ ] 搭建页面路由（重命名 / 格式转换 / 压缩 / 设置）
- [ ] 搭建 Pinia store 骨架
- [ ] 实现基础布局（侧边栏 + 内容区 + 底部状态栏）

---

### 第二阶段：批量重命名模块（5天）

**Day 3-4: 文件导入与列表**

- [ ] 实现文件选择对话框（多选文件 / 选择文件夹）
- [ ] 实现拖拽导入区域（drag & drop）
- [ ] 读取文件信息（名称、大小、格式、尺寸）
- [ ] 文件列表组件（原名称 | 新名称 | 状态）
- [ ] 支持递归子目录导入（可选开关）
- [ ] 文件格式过滤（仅 PNG/JPG/JPEG/WEBP/GIF/HEIC）

**Day 5-6: 命名规则引擎**

- [ ] 实现模板变量系统：`{original}`, `{index}`, `{type}`, `{module}`, `{date}`
- [ ] 类型/模块下拉选择器（product/banner/avatar、homepage/detail/blog）
- [ ] 支持自定义类型和模块
- [ ] 序号格式设置（起始值、步长、位数）
- [ ] 实时预览：左侧原名 → 右侧新名
- [ ] 支持单条修改覆盖

**Day 7: 冲突处理与执行**

- [ ] 冲突检测算法（同名文件检测）
- [ ] 三种策略：自动编号（默认）、覆盖、跳过
- [ ] 输出选项：覆盖原文件（需确认）/ 输出到新目录
- [ ] 保留目录结构选项
- [ ] 执行重命名（fs.rename / fs.copyFile）
- [ ] 错误处理：文件损坏跳过、非法字符自动清理、超长截断

---

### 第三阶段：命名格式转换模块（2天）

**Day 8: 格式转换核心**

- [ ] 实现分词器（支持中英文混合输入）
  - 英文：按空格、驼峰、下划线、连字符分词
  - 中文：简单按字/词分割（V1 先不接 AI 翻译）
- [ ] 实现 6 种格式转换：
  - camelCase
  - PascalCase
  - snake_case
  - SCREAMING_CASE
  - kebab-case
  - package.case

**Day 9: 格式转换 UI**

- [ ] 单行输入 + 即时转换展示
- [ ] 批量模式（多行输入 → 多行输出）
- [ ] 一键复制（单项 / 全部）
- [ ] 历史记录（最近 20 条）

---

### 第四阶段：图片压缩模块（3天）

**Day 10-11: Sharp 集成**

- [ ] 安装 Sharp（注意 Electron rebuild）
- [ ] 实现压缩服务：
  - 质量调节（0-100）
  - 尺寸缩放（百分比）
  - EXIF 去除
  - 格式转换（JPG ↔ WebP ↔ PNG）
- [ ] 批量压缩流水线
- [ ] 压缩前后大小对比展示

**Day 12: 压缩 UI**

- [ ] 参数设置面板（质量滑块、尺寸、格式选择）
- [ ] 压缩预览（前后对比）
- [ ] 批量处理进度
- [ ] 输出目录选择

---

### 第五阶段：任务系统 + 收尾（3天）

**Day 13: 任务队列**

- [ ] 任务队列管理器（并发控制，默认 4 并发）
- [ ] 任务状态：pending → running → done / failed
- [ ] 进度条组件（总进度 + 当前文件）
- [ ] 暂停 / 继续功能
- [ ] 失败重试

**Day 14: 异常处理与优化**

- [ ] 文件损坏检测 → 跳过 + 提示
- [ ] 非图片文件 → 自动过滤
- [ ] 非法字符自动清理（Windows/Mac 文件名限制）
- [ ] 超长文件名截断 + hash 后缀
- [ ] 大批量文件性能优化（虚拟滚动列表）

**Day 15: 打包测试**

- [ ] Windows 打包测试（NSIS 安装包）
- [ ] macOS 打包测试（DMG）
- [ ] 基础功能全流程测试
- [ ] 性能验证：100 张图片 < 10s
- [ ] Bug 修复

---

## V2 开发计划（AI 离线命名）

### 第六阶段（2周）

**Week 1: AI 模型集成**

- [ ] 下载 MarianMT 中英翻译 ONNX 模型
- [ ] 集成 onnxruntime-node
- [ ] 实现中文 → 英文翻译 pipeline
- [ ] 中文分词（nodejieba / segmentit）
- [ ] 后处理规则：小写化、去停用词、替换空格为 `-`、删除特殊字符、长度限制 ≤60

**Week 2: AI 命名 UI + 优化**

- [ ] AI 命名开关（全局 / 单文件）
- [ ] 两种模式：直译 / SEO优化（默认）
- [ ] AI 命名结果预览
- [ ] 重命名模板新增 `{ai}` 变量
- [ ] 翻译失败 fallback（拼音 / 原名）
- [ ] 模型加载性能优化

---
