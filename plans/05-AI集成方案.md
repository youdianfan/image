# AI 集成方案（离线中英翻译 + SEO命名）

## 目标

实现**完全离线**的中文 → 英文翻译，用于图片文件的智能命名。

---

## 技术方案

### 核心组件

| 组件 | 技术 | 说明 |
|------|------|------|
| 推理引擎 | onnxruntime-node | ONNX 模型推理 |
| 翻译模型 | MarianMT (Helsinki-NLP) | 中→英翻译 |
| 中文分词 | segmentit | 纯 JS 中文分词 |
| 后处理 | 自定义规则 | 小写化、去停用词等 |

---

## 模型选择

### MarianMT — Helsinki-NLP/opus-mt-zh-en

- **来源：** Hugging Face (Helsinki-NLP)
- **功能：** 中文 → 英文翻译
- **模型大小：** ~300MB（ONNX 格式）
- **推理速度：** 单句 <200ms（CPU）
- **质量：** 通用翻译质量良好，短文本（文件名级别）效果优秀

### 模型获取与转换

```bash
# 1. 安装转换工具
pip install optimum[exporters] onnx

# 2. 导出为 ONNX 格式
optimum-cli export onnx \
  --model Helsinki-NLP/opus-mt-zh-en \
  --task text2text-generation \
  ./models/marian-mt/

# 产出文件：
# models/marian-mt/
#   ├── encoder_model.onnx     (~150MB)
#   ├── decoder_model.onnx     (~150MB)
#   ├── tokenizer.json
#   ├── source.spm             (sentencepiece 分词模型)
#   ├── target.spm
#   └── config.json
```

### 模型量化（可选，减小体积）

```bash
# INT8 量化，体积减少约 50%
python -m onnxruntime.quantization.quantize \
  --input models/marian-mt/encoder_model.onnx \
  --output models/marian-mt/encoder_model_int8.onnx \
  --quant_format QOperator

# 量化后: ~75MB encoder + ~75MB decoder = ~150MB 总计
```

## 推理流程

```
输入中文 → 分词 → Tokenize → ONNX推理(Encoder→Decoder) → Detokenize → 后处理 → 输出英文
```

### 详细实现

```typescript
// services/ai.service.ts
import * as ort from 'onnxruntime-node'
import path from 'path'

class AiNamingService {
  private encoder: ort.InferenceSession | null = null
  private decoder: ort.InferenceSession | null = null
  private tokenizer: MarianTokenizer | null = null
  private stopWords = new Set(['the', 'a', 'an', 'is', 'are', 'on', 'in', 'at', 'of', 'to', 'and', 'for'])

  // 初始化模型（应用启动时调用一次）
  async initialize(): Promise<void> {
    const modelDir = path.join(process.resourcesPath, 'models', 'marian-mt')

    this.encoder = await ort.InferenceSession.create(
      path.join(modelDir, 'encoder_model.onnx'),
      { executionProviders: ['cpu'] }
    )

    this.decoder = await ort.InferenceSession.create(
      path.join(modelDir, 'decoder_model.onnx'),
      { executionProviders: ['cpu'] }
    )

    this.tokenizer = new MarianTokenizer(modelDir)
  }

  // 翻译单个文本
  async translate(chinese: string): Promise<string> {
    if (!this.encoder || !this.decoder || !this.tokenizer) {
      throw new Error('AI model not initialized')
    }

    // 1. Tokenize
    const inputIds = this.tokenizer.encode(chinese)

    // 2. Encoder
    const encoderOutput = await this.encoder.run({
      input_ids: new ort.Tensor('int64', BigInt64Array.from(inputIds.map(BigInt)), [1, inputIds.length])
    })

    // 3. Decoder (auto-regressive generation)
    const translated = await this.autoRegressiveDecode(encoderOutput)

    // 4. Detokenize
    return this.tokenizer.decode(translated)
  }

  // 生成 SEO 友好的文件名
  async generateFileName(input: string, mode: 'literal' | 'seo' = 'seo'): Promise<string> {
    try {
      // 翻译
      let english = await this.translate(input)

      // 后处理
      english = this.postProcess(english, mode)

      return english
    } catch {
      // fallback：拼音
      return this.fallbackToPinyin(input)
    }
  }

  // 后处理规则
  private postProcess(text: string, mode: 'literal' | 'seo'): string {
    let result = text.toLowerCase()

    // 去除停用词（SEO模式）
    if (mode === 'seo') {
      result = result
        .split(/\s+/)
        .filter(word => !this.stopWords.has(word))
        .join(' ')
    }

    // 替换空格为连字符
    result = result.replace(/\s+/g, '-')

    // 删除特殊字符（只保留字母、数字、连字符）
    result = result.replace(/[^a-z0-9-]/g, '')

    // 去除连续连字符
    result = result.replace(/-{2,}/g, '-')

    // 去除首尾连字符
    result = result.replace(/^-|-$/g, '')

    // 长度限制 ≤ 60 字符
    if (result.length > 60) {
      result = result.substring(0, 60).replace(/-$/, '')
    }

    return result
  }

  // fallback：中文转拼音
  private fallbackToPinyin(chinese: string): string {
    // 使用 pinyin 库作为 fallback
    // npm install pinyin
    const pinyin = require('pinyin')
    return pinyin(chinese, { style: pinyin.STYLE_NORMAL })
      .flat()
      .join('-')
      .substring(0, 60)
  }

  // 批量处理
  async batchGenerate(inputs: string[], mode: 'literal' | 'seo' = 'seo'): Promise<string[]> {
    const results: string[] = []
    for (const input of inputs) {
      results.push(await this.generateFileName(input, mode))
    }
    return results
  }
}

export const aiNamingService = new AiNamingService()
```

## 替代方案：轻量级翻译（不用 ONNX）

如果 MarianMT 模型太大（~300MB），V1 可以先用轻量方案：

### 方案 A：本地词典翻译

```typescript
// 维护一个常用中英对照词典（适合文件命名场景）
const dictionary: Record<string, string> = {
  '白色': 'white', '黑色': 'black', '红色': 'red',
  '猫': 'cat', '狗': 'dog', '鞋': 'shoes',
  '首页': 'homepage', '详情': 'detail', '列表': 'list',
  '产品': 'product', '横幅': 'banner', '头像': 'avatar',
  '背景': 'background', '图标': 'icon', '按钮': 'button',
  // ... 扩展到 1000-2000 常用词
}

function dictionaryTranslate(chinese: string): string {
  const segments = segmentWords(chinese) // 中文分词
  return segments
    .map(word => dictionary[word] || fallbackToPinyin(word))
    .join('-')
}
```

- 优点：体积极小（<1MB），速度极快
- 缺点：覆盖词汇有限，不能处理新词

### 方案 B：Transformers.js（浏览器端 ONNX）

```typescript
// 在 Electron 渲染进程中使用 Transformers.js
import { pipeline } from '@xenova/transformers'

const translator = await pipeline('translation', 'Xenova/opus-mt-zh-en')
const result = await translator('白色的猫在沙发上')
// → "White cat on sofa"
```

- 优点：封装好，API 简洁，自动下载模型
- 缺点：首次加载需下载模型，渲染进程中运行

## 推荐的渐进式集成策略

```
V1.0: 本地词典翻译（零依赖，即时可用）
  ↓
V1.5: Transformers.js（自动下载模型，用户体验好）
  ↓
V2.0: ONNX Runtime + MarianMT（完全离线，性能最优）
  ↓
V3.0: 图片内容识别（CV 模型，基于图片内容命名）
```

这样每个阶段都有可用的翻译功能，逐步提升质量。

## 性能预估

| 方案 | 模型大小 | 单句翻译 | 内存占用 |
|------|---------|---------|---------|
| 本地词典 | <1MB | <1ms | 忽略不计 |
| Transformers.js | ~300MB (缓存) | ~500ms | ~200MB |
| ONNX Runtime | ~300MB (打包) | ~200ms | ~150MB |
| ONNX 量化版 | ~150MB | ~300ms | ~100MB |

## 模型文件分发策略

### 方案 1：打包到安装包中

```yaml
# electron-builder.yml
extraResources:
  - from: models/
    to: models/
```
- 优点：安装即用，完全离线
- 缺点：安装包增大 150-300MB

### 方案 2：首次使用时下载

```typescript
// 首次启动检测模型是否存在，不存在则下载
async function ensureModelExists(): Promise<void> {
  const modelPath = path.join(app.getPath('userData'), 'models', 'marian-mt')
  if (!fs.existsSync(modelPath)) {
    // 从 CDN/GitHub Release 下载
    await downloadModel('https://your-cdn.com/marian-mt.zip', modelPath)
  }
}
```
- 优点：安装包小，按需下载
- 缺点：首次需要网络
